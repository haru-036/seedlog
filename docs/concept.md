# コンセプト

## サービス名

seedlog

## 一行説明

エンジニア学生の「成長ログ」サービス。日々の開発活動を無意識に記録し、必要な時に使えるエピソードとして整理して返してくれる。

---

## 課題

### 表面的な課題

- LTのネタが思いつかない
- 技術記事が書けない
- 就活ESで話せる開発経験がない

### 本質的な課題

日々の学びや気づきを記録していないため、振り返れない。

### なぜ記録しないのか

「よし、動いた。次やろう」で終わってしまう。自分の経験に価値があると思えていないから、記録しようという気持ちが起きない。

### なぜメモできないのか

- 何をメモすべきか判断できない
- 気づいた瞬間にメモする習慣がない

---

## 既存ツールとの違い

NotionAIやClaude Codeで解決できない理由は機能の差ではなく、ターゲットの差。

|          | NotionAI / Claude Code | seedlog                |
| -------- | ---------------------- | ---------------------- |
| 前提     | 記録がある人向け       | 記録できない人向け     |
| 操作     | 自分でまとめる         | 勝手に聞いてくれる     |
| ユーザー | ツール使いこなせる人   | ツール苦手・続かない人 |

---

## ターゲット

プログラミングはしているが、アウトプットや言語化が苦手な学生。

- Discordには常駐しているが、NotionやObsidianは続かない
- 開発中のハマりや解決を「大したことない」と思って流してしまう
- LT会に参加したことはあるが、発表ネタに困っている
- 就活ESで「開発経験を話せ」と言われても何も思い出せない

---

## コアコンセプト

「記録させる」のではなく「勝手に記録される」体験を作る。

```
GitHubにpush（いつも通りの行動）
　↓
AIが「今日このファイル3回修正してたね、何に詰まってた？」と具体的に聞いてくる
　↓
一言答えるだけでログが溜まる
　↓
「LTしたい」「ES書きたい」時に聞くとAIが整理して返してくれる
```

質問が具体的であることが重要。「今日どうだった？」では答えられないが、「今日auth/login.tsを3回修正してたね、何に詰まってた？」なら答えられる。

---

## ユースケース

- **LT登壇**：「LTネタまとめて」と聞くと過去のログからネタ候補を返してくれる
- **技術記事**：「記事ネタまとめて」と聞くと過去のログから自分が苦労したポイントを整理して返してくれる
- **就活ES**：「開発経験をまとめて」と聞くとエピソードを整理してくれる
- **振り返り**：ログを眺めると自分の成長が可視化される

---

## ログの入力方法

GitHubのpushだけでなく、手動でも追加できる。

- **GitHub push** → AIが質問を生成してDiscord DMで送る
- **Discordコマンド** → `/log 今日Dockerのネットワーク設定を理解した`
- **Webアプリ** → 画面から直接メモを追加
